Are you an Advanced User?
If you are using a different MQTT provider than HiveMQ, running a custom Mosquitto broker on a Raspberry Pi, or want to customize the source code, you are encouraged to do so! Simply open the project in VS Code and edit config.json and main.cpp manually. You do not need to proceed with this setup generator.
HiveMQ Details
You need a free MQTT broker to let the lamps talk to each other. Go to HiveMQ Cloud, sign up for a free "Serverless Multi-tenant" cluster, create credentials, and paste them here.

Flashing the Firmware

Now that your `config.json` is ready, it's time to upload the software to the ESP32.

Important: 2-Step Process

In PlatformIO (VS Code), you must do two separate uploads:

1. Build & Upload: Click the main right-arrow at the bottom to flash main.cpp.

2. Upload File System Image: Click the PlatformIO Ant head icon on the left sidebar, open `esp32dev` > `Platform`, and click `Upload File System Image`. This uploads your `config.json`.

If you don't do step 2, your lamp will turn on but it will not connect to Wi-Fi because it won't have the config file!

Hardware Assembly

Now that the brains are programmed, put it all together!

  • 3D Printing: Download the STL files from the GitHub Repo. Print the Base in a solid color, and the Shade in White PLA/PETG for good light diffusion.
  • PCB & Soldering: Order the provided Gerber files from a PCB manufacturer. Solder your components, including the through-hole RGB Common Anode LEDs (diffused/frosted 5mm recommended) into the board.
  • Touch Sensor: Connect the TTP223 module to 3.3V, GND, and GPIO Pin 4. Paste it underneath the top of the lamp shade.
  • Power: Plug a micro-USB or USB-C cable into the ESP32 to power the entire rig.
